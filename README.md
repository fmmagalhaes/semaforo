# Semáforo

A simple implementation of "Jogo do Semáforo" (Alan Parr's Traffic Lights).

## Play

Visit https://fmmagalhaes.github.io/semaforo/

### vs PC

The PC plays opportunistically: it takes any move that wins immediately, and otherwise avoids moves that hand the opponent an immediate win on the next turn. Beyond that, it picks randomly.

### Online (P2P)

Click **Online** to host a game. A shareable link is generated — send it to your opponent and they join as the second player.

Connection is established directly between the two browsers via WebRTC ([PeerJS](https://peerjs.com/)) — there is no game server, only PeerJS's public signalling broker to exchange connection details.

Moves and resets are sent over the data channel; both clients run the same game logic locally and stay in sync.

## Initial Prompt

The first commit was generated by Claude Opus 4.7 using the following prompt:

```
Build a minimalist and modern implementation of "Jogo do Semáforo" (Alan Parr's Traffic Lights) using only vanilla HTML, CSS, and JavaScript.

Requirements:

- Single page application only
- The game starts immediately when the page loads
- No setup screen or menus
- Minimalist UI with clean spacing and large readable elements
- Responsive design that works well on desktop, tablet, and mobile
- Use Flexbox for all layouts
- No frameworks or libraries

Open in the browser when you're done
```

This prompt yielded a working game (local PvP only) in **less than 1 minute.**

Further prompts will not be documented.

### Alternative Prompt

A much shorter prompt such as "Implement Alan Parr's Traffic Lights (3x4)" should also yield a fully functional game. The extra detail mainly shapes the aesthetic and code organisation.
